# KPI Reports

This folder contains user-requested generated KPI dashboards and interactive reports, usually in HTML format.

Use it only for on-demand outputs such as:
- AVEVA E3D project health dashboards.
- Database quality, completeness, or progress KPIs.
- Discipline reports with filters, charts, tables, and drill-down sections.
- Exportable HTML summaries requested by users during project work.

Reports in this folder are generated artifacts, not canonical engineering rules and not the core PML coding contract. If a report reveals a reusable rule, naming convention, project decision, or project standard, file that knowledge back into [project-knowledge/](../project-knowledge/README.md) or the appropriate reference page.

## Demo Report Catalog

- [piping-demo](piping-demo/README.md) — general piping KPI dashboard with component mix, route workload, status, spatial, and QA views.
- [model-health-demo](model-health-demo/README.md) — model health and anomaly dashboard for missing positions, unset flags, missing references, and branch outliers.
- [mto-readiness-demo](mto-readiness-demo/README.md) — construction and material readiness dashboard for shop/field split, spools, welds, flanges, gaskets, valves, and instruments.
- [vault-health-demo](vault-health-demo/README.md) — vault structure dashboard for report package completeness, knowledge clusters, prompts, and reference coverage.

Machine-readable indexes are available as [report-index.json](report-index.json) and [report-index.csv](report-index.csv).

## Report Package Standard

Create one subfolder per report. Each report package should contain:

```text
user-support/kpi-reports/
  piping-progress/
    README.md
    report.json
    piping-progress.html
    piping-progress-data.json
    summary.md
    exports/
      metrics.csv
      anomalies.csv
      route-workload.csv
```

- `README.md` is the Obsidian-facing wrapper and graph node.
- `report.json` is the package manifest for indexing, automation, source traceability, and future regeneration.
- `<report-name>.html` is the interactive dashboard, suitable for histograms, charts, filters, tables, maps, drill-down views, and segmented audience views.
- `<report-name>-data.json` stores parsed source data and KPI summaries used by the dashboard.
- `summary.md` gives a static, Obsidian-native report summary for fast reading, backlinks, and search.
- `exports/*.csv` gives engineer-friendly extracts for Excel, review meetings, or downstream tools.

Use an Obsidian wrapper like this:

```md
# Piping Progress KPI

[Open interactive report](piping-progress.html)

<iframe src="piping-progress.html" width="100%" height="900"></iframe>
```

Keep each package self-contained so the report can be viewed from Obsidian, opened in a browser, linked from project notes, regenerated by an agent, and archived with the exact data that produced it.

## Report Families

Preferred report families include:
- `model-health` — missing attributes, unset flags, invalid references, suspicious coordinates, branch/component outliers.
- `mto-readiness` — material and construction readiness, spool records, shop/field split, weld/flange/gasket/valve counts.
- `progress-tracking` — built flags, status attributes, area/discipline completion, handover readiness.
- `line-list-summary` — pipe, branch, bore, spec, duty, pressure, and temperature rollups.
- `spatial-density` — plan scatter, area congestion, elevation bands, and routing hot spots.
- `qa-qc-checks` — rule violations linked back to [project-knowledge/](../project-knowledge/README.md) standards.
- `vault-health` — vault structure, report package completeness, linkable knowledge clusters, and automation readiness.

## Generation

The bundled demo packages are regenerated by:

```bash
node user-support/kpi-reports/scripts/generate_demo_kpi_reports.js
```

When creating new reports, prefer a repeatable generator over hand-editing generated HTML or data. Update [report-index.json](report-index.json), [report-index.csv](report-index.csv), and the chronological [log.md](../../log.md) entry whenever report packages are added or materially changed.
